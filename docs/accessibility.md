# Accessibility

The game targets keyboard, mouse, touch, and gamepad parity for all primary flows. Menus use semantic DOM controls, logical focus order, visible focus rings, readable labels, and no color-only status. Canvas events are mirrored in a concise HUD and status region without flooding announcements.

Settings include remappable controls, aim sensitivity, reticle style/size/contrast, aim assistance, text scaling, independent volume buses, screen-shake toggle, CRT toggle, reduced motion, reduced flashing, and practice-mode game speed. Pause is always available. Touch targets respect safe areas and remain large enough for phone use.

Reduced motion disables parallax acceleration, camera shake, curvature movement, long transitions, and nonessential vegetation motion. Reduced flashing replaces muzzle/hit flashes with steady outlines and caps rapid luminance change. High contrast changes reticle shape as well as color. Objectives use text, icon, and shape cues.

Accessibility options never prevent campaign progress, achievements based on completion, field-guide discovery, or saving records. Competitive comparisons are local and may record assist configuration for context without stigmatizing it.

Testing covers keyboard-only completion, focus restoration after dialogs, 200% text scale, portrait/landscape safe areas, reduced settings, color-vision-friendly states, touch separation, and TV viewing distance.
